Pequena intro 
Ya teniendo instalado los programas necesarios, ahora vamos a iniciar con la herramienta dbeaver la cual nos permitira gestionar de manera mas grafica nuestras bases de datos la iniciamos y vemos que nos carga demos un pequeno repaso a la misma con al finalidad de entender ciertas cosas. En la parte de arriba tenemos el tipico menu de navegacion  donde haremos foco en el menu windows para poder configurar que las palabras claves pasen a mayusculas de manera automatica, muy util para no preocuparnos por ese tipo de cosas, luego tenemos el menu de accesos rapidos el mismo tiene varios que nos interesa el que dice SQL, este nos permite tanto ejecutar una nueva consulta sql como ver las que estan guardas en el pc, mas a la derecha tenemos dos importantes que ver el primero nos permite ver a que sgdb estamos conectados aqui vamos a ver ejemplo si tenemos conexiones con postgres - mysql - sql server u otro las vamos a ver en este boton de manera rapida y el siguiente nos permite ver las base de datos a la que estamos conectados (cuando seleccinamos el sgdb aparecera un boton para ver las bases de datos que tiene creadas ese gestor). 

Luego el programa se divide en dos parte fundamentales, una donde vemos tanto los gestores como las bases de datos a mano izquierda y el panel principal que esta a mano derecha el mismo cuando tengamos consultas sql abiertas estara de este lado. 


Primera parte
Para generar nuestra primera conexion con dbeaver, podemos hacerlo de varias maneras: 
	1. La primera es con el menu de navegacion con el nombre de database - nueva conexion con base de datos 
	2. Aqui se abrira una ventana en la cual se pueden ver los sgdb mas populares, sql lite - postgre - mariadb - mysql entre otros aqui debemos seleccionar el gestor que vamos a usar en nuestro caso es postgre cuyo logo es un elefante. 
	3. Le damos a next y aqui nos pedira los datos principales para generar una conexion 
		a. Debemos cargar los datos para nuestro caso por ahora dejamos todo como esta 
		b. Solo debemos cargar la clave (esta es la misma que creamos cuando instalamos postgre) 
		c. Por ultimo le damos a finalizar y deberiamos tener acceso a postgre el mismo nos figurara en la parte izquierda ahora como un nuevo gestor de bases de datos. (aqui hacer el ejemplo rapido mostrar todas las bases de datos, ya que es un error comun que pase esto si somos nuevos) 
	4. Mostrar la estructura general de la conexion para que comiencen a comprender ciertas cosas de la interfaz. 

Segunda parte 
Ahora vamos a iniciar a crear nuestra primera base de datos, la idea es hacerlo de dos formar una mas visual otra un poco de codigo para ir coniciendo mas la aplicacion, este punto necesito que hagan tal cual los que les voy pidiendo ya que estas bases de datos seran las que nos sirvan para realizar las practicas mas adelante, asi mismo vamos a ir mostrando el paso a paso y de una manera simple para que puedan comprender todo lo necesario. 

Vamos a crear dos bases de datos al final las pueden dejar ambas o borrar una si desean pero solo para que vean dos formas distintas de generar una base de datos compleja. 

La primera sera de manera visual la cual vamos a hacer lo siguiente 
	1. En la parte izquierda de la aplicacion podemos ver el logo de postgres (un pequeno elefante) hacemos doble click
	2. Vemos la estructura que maneja dividida en bases de datos - administracion - info de sistemas (hacemos click en bases de datos)
	3. Se despliega las bases de datos que tenemos creadas postgre crea una como muetra la misma no tiene datos pero le podemos cargar. 
	4. Encima del menu base de datos hacemos clikc derecho, de despliega el menu y presinamos sobre el que dice crear nueva base de datos - aqui solo debemos poner un nombre y presionamos ok (como primera recomendacion evitar nombres largos y espacios en el nombre de las bases de datos)
	5. Por ultimo en el mismo menu podemos presionar f5 o darle click derecho y darle al menu de refrescar esto para poder ver la nueva base de datos creada.
	6. Si ya podemos ver nuestra base de datos ahora si podemos iniciar la carga de datos correspondiente al archivo universidad_db.sql
	7. La misma se hace la siguiente manera, hacemos click derecho encima de la base de datos creada. 
	8. Buscamos la opcion que dice herramientas o tools en ingles, luego a restaurar - en este cuadro solo debemos buscar nuestro archivo .sql el cual esta como material en este video, se nos abre el gestor de archivos de windows en mi casa aqui debo estar pendiente debido a que por default toma solo archivos .backup pero cambiando la extension podemos ver que tambien reconoce los .sql seleccionamos esto y buscamos la carpeta donde esta el archivo descargado. 
	9. Por ultimo le damos a iniciar y se cargara toda la estructura de la base de datos y los datos necesarios
	10. Podemos revisar la base de datos siempre recordando que debemos actulizar con f5 o click derecho refrescar 

La segunda forma es un poco mas compleja aunque obtendremos el mismo resultado la voy a mostrar como para que se familiaricen con las sentencias sql y los scripts
	1. Descargamos los archivos que estan como material extra del video que se llaman schema.sql y data_ingestion.sql, con estos archivos obtendremos la misma base de datos que con el metodo anterior solo que usando sentencias sql que por ahora no tienen mucho sentido pero mas adelante les llamen la atencion y quieran examinarla ustedes mismos. 
	2. La base de datos la creamos con la siguiente sentencia sql create database name_db presionamos f5 para poder ver nuestra base de datos nueva 
	3. Ahora lo que hacemos es conectarnos a dicha base de datos para que los scripts de estructura y data se carguen dentro de esta. 
	4. Ahora necesitamos hacerlo en el siguiente orden primero el archivo schema.sql (lo mas facil es arrastrar el archivo hasta el programa) 
	5. Este archivo le vamos a dar una vista rapida en la que podemos ver la estructura y relacion entre tablas de toda la base de datos, por ahora no van a enteder mucho de este codigo mas adelante sera mas comprensible para ustedes. 
	6. Este script los ejecutamos con la combinacion de teclado alt + x el cual iniciara el proceso y nos creara la estructura o tablas de la base de datos
	7. Como podemos saber que todo esta de manera correcta, hacemos lo siguiente primero el mensaje de confirmacion que todo esta ok nos indicara que hay 16 modificaciones nuevas eso nos indica la cantidad de tablas que tiene nuestra base de datos. 
	8. Por ultimo presionamos f5 posicionados en la parte izquierda de la aplicacion, o dando click derecho encima de la base  y presionamos refrescar o refresh 
	9. Luego podemos ver nuestras tablas haciendo click en la flecha que esta al lado de nuestra de base de datos - luego en schema - public - tablas (en esta ultima podemos ver las tablas creadas por el script) 
	10. Ya con las tablas creadas podemos cargar el archivo de data_ingestion.sql (arrastrasmos el archivo hasta el programa) 
	11. Alli podremos ver todos los scripts con los insert de los datos y procedemos a ejecutar dicho script con las combinacion de teclas alt + x 
	12. En este caso pasa lo mismo la ventana de mensajes nos indica que se cargaron en 16 tablas los datos correspondientes. 
	13. Podemos verificar de manera visual los datos presionando en algunas de las tablas y buscando la opcion de data. 

En este punto podemos ver algunas datos en las tablas y ver tambien el schema, enfatizar que este programa es muy bueno generando esto ya que otros es mas complicado verlo aunque existen opciones mejores a nivel de interfaz como navicat pero son de pago y muy costosas.
